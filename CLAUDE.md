# CLAUDE.md

This file provides guidance to Claude Code (`claude.ai/code`) when working with code in this repository.

## Project Overview

This repository is a static GitHub Pages site about German elections and polling. It now has three user-facing pages under `docs/`:

- `docs/index.html` — federal polling trends from Forschungsgruppe Wahlen / Politbarometer
- `docs/demographics.html` — historical Bundestag demographic voting patterns from Bundeswahlleiterin Heft 4
- `docs/state-polling.html` — Bundesland left-vs-right polling map and table from `dawum.de`

There is no frontend build step and no npm toolchain. The site is plain HTML/CSS/ES modules, with committed JSON data files under `docs/data/`.

## Commands

Set up the local Python environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Refresh federal polling data:

```bash
python3 scripts/fetch_data.py
```

Refresh state polling data:

```bash
python3 scripts/fetch_state_data.py
```

Re-extract demographics data from the Bundeswahlleiterin PDF:

```bash
python3 scripts/extract_demographics.py
```

Run a local dev server:

```bash
python3 -m http.server -d docs
```

There are currently no tests, linters, or build commands.

## Deployment And Automation

- The site is structured for GitHub Pages from `docs/`.
- `.github/workflows/update-data.yml` runs daily at `06:00 UTC` and also supports manual dispatch.
- The workflow uses Python `3.12`, installs `httpx`, `pandas`, and `openpyxl`, then runs:
  - `python scripts/fetch_data.py`
  - `python scripts/fetch_state_data.py`
- If data changed, it commits:
  - `docs/data/polling.json`
  - `docs/data/.etag`
  - `docs/data/state-polling.json`
  - `docs/data/state-polling.last_update.txt`

`docs/data/pre-polling.json` is committed to the repo and maintained separately from the daily GitHub Action. `scripts/fetch_data.py` reads it and prepends those rows when writing `docs/data/polling.json`.

The demographics JSON is committed to the repo but is not regenerated by the GitHub Action.

## Data Pipelines

### 1. Federal polling (`scripts/fetch_data.py`)

- Downloads the Excel workbook from Forschungsgruppe Wahlen:
  - `https://www.forschungsgruppe.de/.../1_Projektion_1.xlsx`
- Uses ETag caching via `docs/data/.etag`; on `304 Not Modified`, it still checks whether existing `docs/data/polling.json` needs the pre-polling rows prepended.
- Reads sheet `Tabelle1` with `skiprows=9`.
- Assumes:
  - column `0` is empty
  - column `1` is the date
  - columns `2-10` are party values
- Maps Excel headers to the JSON keys:
  - `cdu`, `spd`, `gruene`, `fdp`, `linke`, `piraten`, `afd`, `fw`, `bsw`
- Computes `sonstige` as `100 - sum(other parties)` and clips it at `0`.
- Prepends committed historical rows from `docs/data/pre-polling.json`, filtering out rows that overlap the fetched Excel series.
- Writes `docs/data/polling.json` with top-level shape:
  - `updated`
  - `data: [{ date, cdu, spd, ... }]`

Important implementation detail: the script uses `StrEnum`, so local runs expect Python `3.11+`.

### 2. State polling (`scripts/fetch_state_data.py`)

- Fetches:
  - `https://api.dawum.de/newest_surveys.json`
  - `https://api.dawum.de/last_update.txt`
- Uses `docs/data/state-polling.last_update.txt` as a lightweight change detector.
- Computes a weighted average per Bundesland across recent state surveys:
  - weight = `sqrt(sample_size) * 0.5^(age_days/30)`
- Defines:
  - left = `SPD + Grüne + Linke`
  - right = `CDU/CSU + AfD`
- Falls back to `scripts/state_election_results.json` if a state has no current surveys.
- Writes `docs/data/state-polling.json` with:
  - `updated`
  - `source`
  - `summary`
  - `states`

The state page also depends on the committed map asset `docs/data/germany-states.svg`.

### 3. Demographics (`scripts/extract_demographics.py`)

- One-off/manual extraction script that downloads `btw25_heft4.pdf`.
- Parses Bundeswahlleiterin Heft 4, pages `15-23` (0-indexed page slice `14..22`).
- Extracts "Übersicht 9: Zweitstimmen nach Geschlecht und Altersgruppen seit 1953".
- Writes `docs/data/demographics.json`.
- Handles:
  - normalized age-bracket keys
  - gender buckets `insgesamt`, `frauen`, `maenner`
  - CSU embedded in CDU for `1953` and `1957`

Treat `docs/data/demographics.json` as the frontend source of truth unless you intentionally want to regenerate it from the PDF.

## Frontend Architecture

### Shared

- `docs/css/style.css` contains the shared styling for all three pages.
- `docs/js/config.js` is the shared federal-party config:
  - labels
  - colors
  - block membership
  - election marker dates
  - EWMA halflife (`30` days)
- `docs/fonts/` and `docs/js/vendor/plotly.min.js` are self-hosted.

### Federal polling page

Entry point and modules:

- `docs/index.html`
- `docs/js/main.js`
- `docs/js/data.js`
- `docs/js/charts.js`
- `docs/js/config.js`

Behavior:

- Loads `docs/data/polling.json`
- Converts date strings to `Date`
- Computes block totals dynamically from `PARTY_CONFIG.block`
- Renders two Plotly charts:
  - major parties
  - political blocks
- Applies EWMA smoothing with a 30-day halflife
- Shows election markers from `ELECTION_DATES`
- Uses mobile/desktop layout differences at `768px`

`MAJOR_PARTIES` in `docs/js/config.js` controls which parties appear in the party chart.

### Demographics page

Entry point and modules:

- `docs/demographics.html`
- `docs/js/demographics-main.js`
- `docs/js/demographics-data.js`
- `docs/js/demographics-charts.js`
- `docs/js/demographics-config.js`

Behavior:

- Loads `docs/data/demographics.json`
- Lets the user build up to `5` comparison lines
- Each line combines:
  - gender
  - age bracket
  - one or more parties
- `union` is a virtual frontend-only party combining `CDU + CSU`
- Missing age buckets create chart gaps; missing party values within an existing row are treated as `0`
- Renders a single Plotly line chart with configurable selections

### State polling page

Entry point and modules:

- `docs/state-polling.html`
- `docs/js/state-polling-main.js`
- `docs/js/state-polling-data.js`
- `docs/js/state-polling-charts.js`
- `docs/js/state-polling-config.js`

Behavior:

- Loads `docs/data/state-polling.json`
- Fetches and inlines `docs/data/germany-states.svg`
- Renders:
  - a summary segment bar
  - a current-diff map
  - a change-vs-election map
  - a sortable table
- Uses direct DOM/SVG manipulation rather than Plotly for the maps
- Colors are derived from a diverging scale centered at `0`, with config in `state-polling-config.js`
